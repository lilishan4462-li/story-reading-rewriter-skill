# WorkBuddy 适配指南

## 适配目标

让 WorkBuddy 能读取并执行专业版规则，把一本难读书改写成连续追剧式知识小说。

## 推荐接入方式

优先把这些文件加入 WorkBuddy 的项目知识、工作区文件或长期上下文：

- `professional/SKILL.md`
- `professional/README.md`
- `professional/references/rewrite-rules.md`

如果 WorkBuddy 支持项目规则，把核心要求写入项目规则：

```text
你正在使用 story-reading-rewriter 方法。
所有改写任务必须先建立内容大纲，再逐章改写。
全书必须保持同一个主角、同一个世界、同一条主线。
每章只承载一个知识关卡。
知识必须从剧情冲突、选择、代价和后果中显现。
每章故事后保留简短知识揭底。
失败章节只做定向修复，不随意重开世界线。
```

## 第一次运行流程

1. 新建一个独立项目或会话。
2. 上传或粘贴专业版规则。
3. 提供书籍目录和第一章正文。
4. 先要求 WorkBuddy 输出内容大纲和前 5 章知识关卡。
5. 只试写第 1 章。
6. 使用质量闸门检查第 1 章。
7. 通过后再继续第 2 章。
