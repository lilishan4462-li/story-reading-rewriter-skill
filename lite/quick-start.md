# 追剧式读书改写器 Lite 快速开始

## 适合场景

你使用的是豆包、DeepSeek、Kimi、通义、普通 ChatGPT 这类对话框软件，而不是 Codex、Claude Code、Cursor 这类助手工具。

## 三步开始

### 第一步：上传规则包

把 `story-reading-rewriter-lite.md` 上传给大模型，或直接粘贴进去。

然后发送：

```text
请读取这个规则包。读完后，你来主持整本书的追剧式改写流程。
```

### 第二步：上传书籍材料

优先上传整本书。

如果平台不支持整本读取，就提供：

1. 书名和作者；
2. 目录；
3. 第一章正文；
4. 你希望的故事类型。

### 第三步：让模型开始主持

发送：

```text
开始。请先判断正文材料，再建立内容大纲。不要直接写第一章。
```

## 正确的模型行为

模型应该先做：

1. 材料判断；
2. 内容大纲；
3. 前 5 章章节计划；
4. 第 1 章试写；
5. 质量闸门；
6. 必要时定向修复；
7. 输出项目状态卡。

如果模型一上来就写正文，说明它没有遵守规则。你可以回复：

```text
停止。请回到追剧式读书改写器 Lite 流程。
先建立内容大纲，再写正文。
```

## 换对话继续

每次结束前，复制模型输出的“项目状态卡”。

新开对话时，先发送：

```text
请继续这个追剧式读书改写项目。下面是项目状态卡，请先恢复上下文，不要重开设定。
```

然后粘贴状态卡。
